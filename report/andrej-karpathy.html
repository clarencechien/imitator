軟體三世代

Karpathy 將軟體演進劃分為三個截然不同的世代。這不僅是技術的迭代，更是程式設計思維的根本性轉變。從明確的指令到訓練權重，再到用自然語言編程，每一次變革都開啟了新的可能性。

💻

軟體 1.0

傳統程式碼

由工程師明確撰寫給電腦的指令，如 Python、C++。這是我們所熟知的數位世界基石。

🧠

軟體 2.0

神經網路權重

透過大量資料訓練出的參數。工程師的角色轉變為資料的策展人與優化者。

💬

軟體 3.0

用提示編程

用自然語言（如英文）指揮 LLM。Karpathy 稱之為「最熱門的新程式語言」。

2.0 吞噬 1.0：特斯拉 Autopilot 案例

在特斯拉，神經網路（軟體 2.0）逐漸取代了傳統 C++ 程式碼（軟體 1.0），承擔了越來越複雜的整合與決策功能。下方的圖表模擬了這個演進過程。

解構 LLM

要善用 LLM，我們必須理解它的本質。Karpathy 提供了多個角度，從它在生態系中的角色，到它奇特的「心理」特質，幫助我們建立一個更準確的心智模型。

🔌 像「公共設施」

提供如電力般的標準化智能服務。我們按用量付費，並要求其穩定可靠。當它當機時，世界便會經歷「智力停電」。

🏭 像「晶圓廠」

訓練 LLM 需要極高的資本與技術壁壘，如同半導體製造，技術集中在少數實驗室手中。

🖥️ 像「作業系統」

最貼切的比喻。它是一個複雜的生態系，而非單純商品。我們正處於其發展的「大型主機年代」，透過終端（聊天介面）與之互動。

LLM 如同一個有認知缺陷的天才，理解其優缺點是與之協作的關鍵。

🏆

超能力：百科全書記憶

像電影《雨人》主角，擁有近乎完美的記憶力。

👻

缺陷：幻覺

會捏造事實，缺乏對自身知識的內部模型。

📉

缺陷：參差不齊的智能

在難題上表現超凡，卻可能在簡單問題上犯錯。

🧠

缺陷：順行性遺忘症

像電影《記憶拼圖》，無法持續學習，每次對話都像重啟。

AI 時代的機會

新的軟體典範帶來了巨大的機會。Karpathy 指出了三個主要方向，從革新現有軟體到創造全新的互動模式，這些都是創業者和開發者可以大展拳腳的領域。

打造「鋼鐵人戰衣」，而非機器人

成功的 LLM 應用（如 Cursor）是人類的增強工具，而非完全取代。關鍵在於加速「生成-驗證」循環，並提供一個「自動化滑桿」讓使用者掌控一切。

人機協作循環

🤖 AI 生成 ➔ 🧑‍💻 人類驗證 ➔ 🔁 迭代

目標：讓此循環越快越好，並將 AI 保持在「可控範圍」。

Vibe Coding：人人都是開發者

當程式語言是英文時，任何人都能憑「感覺」創造軟體。這是通往軟體開發的「入門藥」。

Karpathy 的 MenuGen.app 經驗：

  • 簡單的部分：用 Vibe Coding 快速完成核心功能。
  • 困難的部分：部署、域名、金流、驗證...這些專為人類點擊設計的 DevOps 流程。

為代理人 (Agents) 而建構

數位世界將迎來新的消費者：AI 代理人。我們需要為它們打造專屬的基礎設施，讓它們能更高效地理解和操作我們的軟體。

之前 (為人類)

文件充滿視覺元素和模糊指令，如：「點擊這裡」來部署。

未來 (為代理人)

提供 Markdown 文件和可執行的 `cURL` 指令，讓代理人能直接操作。

未來展望：推動自動化滑桿

Karpathy 總結道，我們正處於一個需要大量重寫和創造新軟體的時代。未來的十年，我們的工作就是不斷推動「自動化滑桿」，從「增強工具」逐漸走向「自主代理人」。這是一個充滿挑戰與無限機會的旅程。

🧑‍🔧 增強 (Augmentation) 🤖 代理人 (Agent)

我們目前在這裡：打造更強大的「鋼鐵人戰衣」。
