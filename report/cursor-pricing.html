Cursor 收費方案詳解

了解 Cursor 的計價方式，聰明選擇最適合您的方案。

Cursor 目前方案與 Request 計價

方案 月費 每月高速額度 其他重點
Hobby 免費 50 Request / 200 補全 含 14 天 Pro 試用
Pro 20 USD 500 Request + 無限 Slow Request 可使用 Max Mode
Business 40 USD/人 與 Pro 同 可強制隱私模式、SSO、集中帳單

統一價：1 Request＝0.04 USD

不論選哪個 premium 模型，在 Normal Mode 都只按「次數」計數，不看 Token。

免費額度

用完高速額度後，自動降速 (Slow)，雖然排隊時間可能較久，但不另外收費。

付費外掛

若綁自家 OpenAI/Anthropic API Key，可跳過 Cursor 計費，直接照官方 Token 價，整體更低。

Max Mode 計價與適用時機

計價公式

Token 數 × (模型官方 API 價格 × 1.2)

Cursor 表示固定加 20% 服務費。

Token 類型

  • Input：新 Token
  • Input (Cached)：舊 Token
  • Output：模型回覆，依類型分階梯計價

對照 Normal Mode

  • Normal：固定 Request，適合 99% 日常問答、單檔修改。
  • Max：200 Tool Calls、1M Token 視窗，適合全倉庫重構、大型推理、複雜 Agent 工作。

成本示例 (Claude 3.7 Sonnet)：

每 Prompt 約 0.05 USD + 每次 Tool Call 約 0.05 USD。

❗ 若不控管可一次噴 10 USD。

省錢策略總結

  • 優先用 Auto-select + Slow Request：讓 Cursor 自動挑便宜可靠模型，超過 500 Fast 額度後仍免費排隊。
  • 只在真正需要時開 Max Mode：視窗很大但每 1M Token 至少數美元，讓它只處理必要檔案；否則拆分任務、分段對話。
  • 用自己的 API Key：OpenAI/Anthropic 官方 Token 價通常 <0.04 USD/Req；綁定後 Cursor 僅做前端，成本可降至 1/3–1/5。
  • 控管上下文：
    • 移除註解、大檔案只貼片段，可省 30–70% Token。
    • 利用 @filename 精準載入程式碼，避免整庫上傳。
  • 選擇輕量模型起手，必要時升級：Debug trace 先用 Gemini Flash 或 Grok Mini，確認問題點，再交給 GPT-4o 完成修補。
  • 定期檢查「Usage Dashboard」：Cursor 提供 Request 與 Token 分布圖，針對異常高峰作優化。
  • 善用社群最佳實踐：論壇有不少降低 Token、加速回覆的腳本與設定檔可以直接套用。

行動建議

日常開發：

把模型鎖在 GPT-4o / Claude 3.5 + Auto-select，確保 500 Fast 內用好用滿。

大型重構或複雜推理：

啟用 Max Mode + Claude 4 Sonnet，並一次性處理最難部分後，切回 Normal。

草稿與搜尋：

Gemini Flash 或 Grok Mini 免費解答，減少 Request 用量。

成本監控：

若月中發現 Request 將爆表，改用 Slow Queue + 自己 API Key；真正要「燒錢」的分析再開 Max 模式。

遵循以上原則，可以在不犧牲模型質量的前提下，把 Cursor 的費用控制在可預期且最低的範圍內。
