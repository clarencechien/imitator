智慧的經濟學：好模型,真的比較省嗎?

Cursor 首份開發者習慣報告,用真實的 agent 用量、token 消耗與 PR 資料,量測「能力 vs 成本」的取捨。以下用它的數據檢驗一個直覺:用越好的模型少犯錯,反而更省錢;用低階模型反覆試錯才是浪費。

假設 / Hypothesis

「好模型 → 少試錯 → 更省;便宜模型 → 一直試錯 → 更貴。」

用三個面向檢驗:每次請求成本、每行被採用程式碼成本、以及成本-品質前緣。

判讀
方向對,結論要修正

方向成立——貴的模型把錢花在「被採用的產出」上,單行成本差距會收斂。
但真正省錢的關鍵不是「越貴」,而是「越靠近成本-品質前緣」。能力不足、不在前緣上的便宜模型,才會用試錯把錢燒掉。

最貴的 Opus 4.7（$1.57）是最便宜 Composer 2.5（$0.18）的近 9 倍。同一套流程,換模型成本天差地遠。

每行「被採用」程式碼成本:差距收斂到 ≈7×

改看「被接受的產出」,差距從 9× 縮到 7×——貴模型靠「更高採用率、更少返工」把溢價賺回一部分。這就是假設成立的那一半。

03

給操作者的三個結論

怎麼選模型才真的省
選「前緣」,不是選「最貴」

Composer 2.5 用 $0.55/任務拿到 63.2%,壓過 Opus 4.7 high（$5.01 / 59.4%)與全部 Sonnet 檔位。被調校好的便宜專用模型,可以同時最便宜又接近最高分。

真正貴的是「能力不足的試錯」

該避免的不是「便宜」,而是「分數太低、不在前緣上」的模型(如 Kimi 2.5 $0.87 → 31.9%):它逼你反覆重來,把錢花在返工而非產出。用「每行被採用成本」而非「每次請求」來算帳。

頂規溢價,報酬遞減

Opus 4.7 要到 max（$11/任務)才拿下 64.8%,只比 GPT-5.5 extra-high（$4.37 / 64.3%)高 0.5 分,卻貴 2.5 倍。最高分留給「錯誤代價極高」的關鍵任務,日常用前緣模型。
